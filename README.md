# QAアーキテクチャ設計思想

本リポジトリは、LLM（ChatGPT等）を  
「プロンプト操作の対象」ではなく  
**確率構造として扱うための分解ログ**である。

目的は、AIを「賢そうに」ではなく、  
**再現性を持って正しく動作させること** にある。

---

## 全体構造

本資料群は、次の順で読むことを推奨する。

### 01_AI動作原理_確率空間モデル
AIは何をしているのかを説明する。
AIは理解していない。確率空間から尤もらしい候補を選んでいるだけである。

### 02_AI入力構造_三層モデル
プロンプト / Knowledge / RAG の役割を分離して整理する。

### 03_プロンプト設計基本構造
安定した出力を得るためのプロンプト設計要素を定義する。

### 04_プロンプト初期条件理論
プロンプトが「最初だけでよい」理由を確率空間の観点で説明する。

### 05_QAチャット運用思想
QAチャットの正しい使い方と誤用を定義する。

### 06_QA行動制約仕様
推測・補完を禁止し、再現性を優先するための行動制約。

### 07_人資料_AI資料_分離設計
人向け資料とAI向け資料を分離すべき理由と判断基準。

---

## 基本思想

- AIは理解していない
- AIは確率的に選択している
- ハルシネーションは設計不備の結果である
- 制御はAIの責任ではなく設計者の責任である

---

## ゴール

- 推測させない
- 判断させない
- 根拠がなければ答えさせない
- 再現性と説明責任を優先する

---

本リポジトリは、  
QAチャットを安全に設計するための「確率空間制御設計思想」である。
