# QAアーキテクチャ設計思想

本リポジトリは、LLM（ChatGPT等）を  
「プロンプト操作の対象」ではなく  
**確率構造として扱うための分解ログ**である。

目的は、AIを「賢そうに」ではなく、  
**再現性を持って正しく動作させること** にある。

---

## 基本思想

- AIは理解していない
- AIは確率的に選択している
- ハルシネーションは設計不備の結果である
- 制御はAIの責任ではなく設計者の責任である

---

## ゴール

- 推測させない
- 判断させない
- 根拠がなければ答えさせない
- 再現性と説明責任を優先する

---

本リポジトリは、QAチャットを安全に設計するための「確率空間制御設計思想」である。
